# [SQLGlot](https://github.com/tobymao/sqlglot/blob/main/README.md) への貢献

SQLGLot はオープンソースソフトウェアです。私たちはフィードバックを重視しており、このプロジェクトへの貢献を
可能な限り容易かつ透明性の高いものにしたいと考えています。具体的には以下の方法で貢献できます。

- バグの報告
- コードの現状に関する議論
- 修正の提出
- 新機能の提案

## 私たちはGitHubを使って開発を行っています。

GitHubはコードのホスティング、問題や機能リクエストの追跡、プルリクエストの受け入れに利用しています。

## 取り組むタスクを見つける

SQLGlot コアチームが問題への対応を予定していない場合、通常は「計画なし」としてクローズされます。これは、リクエストへの対応が非常に困難な場合や、チームが優先すべきではないと判断した場合などに発生します。

これらの問題は、取り組むタスクを探す際の良い出発点となります。問題リストをフィルタリングしてクローズ済みの問題を取得し、「計画なし」とマークされている問題を検索してください。問題の範囲が明確でない場合や、ガイダンスが必要な場合は、遠慮なくお問い合わせください。

タスクを引き受ける前に、[AST 入門](https://github.com/tobymao/sqlglot/blob/main/posts/ast_primer.md) と [オンボーディングドキュメント](https://github.com/tobymao/sqlglot/blob/main/posts/onboarding.md) を熟読することを検討してください。

## コード変更の提出

プルリクエストはコードベースへの変更を提案する最良の方法であり、積極的に歓迎しています。

プルリクエストは小規模で、プロジェクトの慣例に従う必要があります。多くの変更を必要とする機能については、リクエストを行う前に [Slack](https://tobikodata.com/slack) でご連絡ください。
関連するコンテキストを共有し、マージされる可能性を高めます。

1. リポジトリをフォークし、`main` からブランチを作成します。
2. 重要な変更を含むコードを追加した場合は、テストを追加します。
3. API を変更した場合は、ドキュメント (docstring) を更新します。
4. テストスイートとリンターの [チェック](https://github.com/tobymao/sqlglot/blob/main/README.md#run-tests-and-lint) がパスすることを確認します。
5. プルリクエストを発行し、メンテナーまたはコントリビューターによるレビューを待ちます。

注: PR を作成する際は、[Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) のガイドラインに従ってください。

## IMPORTANT: PR の範囲は最小限に抑えましょう

各プルリクエストは、明確に定義された単一の変更に焦点を当てるべきです。複数の無関係な修正や機能を 1 つの PR にまとめるのは避けましょう。これにより、コードレビューの速度と効率が向上し、承認の可能性が高まり、Git の履歴を整理しやすくなります。

## GitHub の [issues](https://github.com/tobymao/sqlglot/issues) を使ってバグを報告してください。

GitHub の Issues は、公開されているバグを追跡するために使用されています。バグを報告するには、新しい Issue を開いてください。

**優れたバグ報告** には、次のような内容が含まれている傾向があります。

- 簡単な概要または背景
- 再現手順
- 具体的な内容
- 可能であればサンプルコード
- 想定していた動作
- 実際に発生した動作
- メモ（発生原因や、試してみたがうまくいかなかった点など）
- 参考資料（例：問題に関連するドキュメントページ）

## GitHub の [discussions](https://github.com/tobymao/sqlglot/discussions) を使ってディスカッションを開始してください。

[GitHub のディスカッション](https://github.com/tobymao/sqlglot/discussions/190) を利用して、コードの現状について議論しています。
新しい機能を提案したい場合は、ここが最適です。ディスカッションを開始し、なぜこの機能が SQLGlot に追加されるべきだと思うのか（使用例なども含めて）をお知らせください。
